In questo articolo vengono forniti esempio di configurazione per la configurazione di Azure macchine Virtuali di estensioni per Windows macchine virtuali.

Per ulteriori informazioni su queste estensioni, vedere [Cenni preliminari sulle estensioni di Azure macchine Virtuali.](virtual-machines-windows-extensions-features.md)

Per ulteriori informazioni sulla creazione di modelli di estensione, vedere [creazione di modelli di estensione.](virtual-machines-windows-extensions-authoring-templates.md)

## <a name="sample-template-snippet-for-vm-extensions-with-iaas-vms"></a>Frammento di modello di esempio per le estensioni macchine Virtuali con macchine virtuali IaaS.
Frammento di modello per la distribuzione ha un aspetto simile estensioni come segue:

      {
      "type": "Microsoft.Compute/virtualMachines/extensions",
      "name": "MyExtension",
      "apiVersion": "2015-05-01-preview",
      "location": "[parameters('location')]",
      "dependsOn": ["[concat('Microsoft.Compute/virtualMachines/',parameters('vmName'))]"],
      "properties":
      {
      "publisher": "Publisher Namespace",
      "type": "extension Name",
      "typeHandlerVersion": "extension version",
      "autoUpgradeMinorVersion":true,
      "settings": {
      // Extension specific configuration goes in here.
      }
      }
      }

### <a name="customscript-extension-14"></a>Estensione CustomScript 1.4.
      {
          "publisher": "Microsoft.Compute",
          "type": "CustomScriptExtension",
          "typeHandlerVersion": "1.4",
          "settings": {
              "fileUris": [
                  "http: //Yourstorageaccount.blob.core.windows.net/customscriptfiles/start.ps1"
              ],
              "commandToExecute": "powershell.exe -ExecutionPolicy Unrestricted -start.ps1"
          },
          "protectedSettings": {
            "storageAccountName": "yourStorageAccountName",
            "storageAccountKey": "yourStorageAccountKey"
          }
      }

#### <a name="parameter-description"></a>Descrizione del parametro:

- fileUris: elenco separati virgola degli URL di file scaricati nella macchina virtuale dall'estensione. Nessun file vengono scaricati se non viene specificato. Se i file si trovano in archiviazione Azure, il fileURLs possibile contrassegnare come privato e la correspoding storageAccountName e storageAccountKey possono essere passati come parametri privati per accedere a questi file.
- commandToExecute: [parametro obbligatorio]: si tratta di comando che verrà eseguito dall'estensione.
- storageAccountName: [parametri facoltativi]: nome dell'Account di archiviazione per l'accesso a fileURLs, se sono contrassegnati come privati.
- storageAccountKey: [parametri facoltativi]: chiave Account lo spazio di archiviazione per l'accesso a fileURLs, se sono contrassegnati come privati.

### <a name="customscript-extension-17"></a>Estensione CustomScript 1.7.

Fare riferimento CustomScript versione 1.4 per descrizione del parametro. Versione 1.7 introduce il supporto per l'invio di script parameters(commandToExecute) come protectedSettings, nel qual caso verranno crittografati prima dell'invio. parametro 'commandToExecute' può essere specificato nelle impostazioni o protectedSettings ma non in entrambi.

        {
            "publisher": "Microsoft.Compute",
            "type": "CustomScriptExtension",
            "typeHandlerVersion": "1.7",
            "settings": {
                "fileUris": [
                    "http: //Yourstorageaccount.blob.core.windows.net/customscriptfiles/start.ps1"
                ],
                "commandToExecute": "powershell.exe -ExecutionPolicy Unrestricted -start.ps1"
            },
            "protectedSettings": {
              "commandToExecute": "powershell.exe -ExecutionPolicy Unrestricted -start.ps1",
              "storageAccountName": "yourStorageAccountName",
              "storageAccountKey": "yourStorageAccountKey"
            }
        }

### <a name="dsc-extension"></a>Estensione DSC.
      {
          "publisher": "Microsoft.Powershell",
          "type": "DSC",
          "typeHandlerVersion": "2.1(Recommendation is to use the latest version)",
          "settings": {
              "ModulesUrl": "https://UrlToZipContainingConfigurationScript.ps1.zip",
              "SasToken": "Optional : SAS Token if ModulesUrl points to Azure Blob Storage",
              "ConfigurationFunction": "ConfigurationScript.ps1\\ConfigurationFunction",
              "Properties": {
                  "ParameterToConfigurationFunction1": "Value1",
                  "ParameterToConfigurationFunction2": "Value2",
                  "ParameterOfTypePSCredential1": {
                      "UserName": "UsernameValue1",
                      "Password": "PrivateSettingsRef:Key1(Value is a reference to a member of the Items object in the protected settings)"
                  },
                  "ParameterOfTypePSCredential2": {
                      "UserName": "UsernameValue2",
                      "Password": "PrivateSettingsRef:Key2"
                  }
              }
          },
          "protectedSettings": {
              "Items": {
                  "Key1": "PasswordValue1",
                  "Key2": "PasswordValue2"
              },
              "DataBlobUri": "optional : https: //UrlToConfigurationData.psd1"
          }
      }

### <a name="azure-diagnostics"></a>Diagnostica Windows Azure

Per ulteriori informazioni su come configurare la diagnostica, vedere [Estensione diagnostica Azure](virtual-machines-windows-extensions-diagnostics-template.md)

          {
            "publisher": "Microsoft.Azure.Diagnostics",
            "type": "IaaSDiagnostics",
            "typeHandlerVersion": "1.5",
            "autoUpgradeMinorVersion": true,
            "settings": {
              "xmlCfg": "[base64(variables('wadcfgx'))]",
              "storageAccount": "[parameters('diagnosticsStorageAccount')]"
            },
            "protectedSettings": {
            "storageAccountName": "[parameters('diagnosticsStorageAccount')]",
            "storageAccountKey": "[listkeys(variables('accountid'), '2015-05-01-preview').key1]",
            "storageAccountEndPoint": "https://core.windows.net"
          }
          }

Negli esempi precedenti, sostituire il numero di versione con il numero di versione più recente.
