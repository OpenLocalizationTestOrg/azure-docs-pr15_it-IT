#<a name="media-services-playready-license-template-overview"></a>Informazioni generali sui modelli di licenza PlayReady servizi multimediali

Servizi multimediali di Azure offre un servizio per l'esecuzione di licenze Microsoft PlayReady. Quando il lettore di utente finale (ad esempio Silverlight) tenta di riprodurre il contenuto PlayReady protetto, viene inviata una richiesta al servizio di recapito licenza per ottenere una licenza. Se il servizio licenze approva la richiesta, problemi alla licenza che viene inviata al client e può essere utilizzata per decrittografare e riprodurre il contenuto specificato.

Servizi multimediali vengono forniti anche API che consentono di configurare le licenze PlayReady. Licenze contengono i diritti e restrizioni desiderate per il runtime PlayReady DRM per imporre quando un utente sta tentando di riprodurre contenuto protetto.
Di seguito sono riportati alcuni esempi di restrizioni di licenza PlayReady che è possibile specificare:

- DateTime da cui la licenza non è valida.
- Valore DateTime quando scade la licenza. 
- Relativa alla licenza da salvare nell'archivio permanente sul client. Licenze persistente in genere utilizzate per consentire la riproduzione offline del contenuto.
- Il livello di protezione minimo che deve disporre un lettore di riprodurre il contenuto. 
- Il livello di protezione di output per i controlli di output per il contenuto di audio\video. 
- Per ulteriori informazioni, vedere la sezione controlli di Output (3,5) nel documento [Le regole di conformità PlayReady](https://www.microsoft.com/playready/licensing/compliance/) .

>[AZURE.NOTE]Attualmente, è possibile configurare solo PlayRight della licenza PlayReady (questo diritto è necessario). Il PlayRight consente il client per la riproduzione del contenuto. Il PlayRight è inoltre possibile configurare le restrizioni specifiche per la riproduzione. Per ulteriori informazioni, vedere [PlayReadyPlayRight](media-services-playready-license-template-overview.md#PlayReadyPlayRight).

Per configurare licenze PlayReady tramite servizi multimediali, è necessario configurare il modello di licenza PlayReady di servizi di supporto. Il modello è definito in XML.

Nell'esempio seguente mostra il modello più semplice (e più comune) che consente di configurare una licenza di flusso base. Con una licenza, i clienti sarà in grado di riproduzione il PlayReady contenuto protetto.
    
    <?xml version="1.0" encoding="utf-8"?>
    <PlayReadyLicenseResponseTemplate xmlns:i="http://www.w3.org/2001/XMLSchema-instance" 
                                      xmlns="http://schemas.microsoft.com/Azure/MediaServices/KeyDelivery/PlayReadyTemplate/v1">
      <LicenseTemplates>
        <PlayReadyLicenseTemplate>
          <ContentKey i:type="ContentEncryptionKeyFromHeader" />
          <PlayRight />
        </PlayReadyLicenseTemplate>
      </LicenseTemplates>
    </PlayReadyLicenseResponseTemplate>

Il codice XML conforme allo schema XML del modello di licenza PlayReady, definito nel modello di licenza PlayReady sezione dello schema XML.

Servizi multimediali definisce anche un insieme di classi .NET che può essere utilizzato per serializzato e deserializzato da e verso il codice XML. Per descrizione delle classi principali, vedere [le classi .NET servizi multimediali](media-services-playready-license-template-overview.md#classes). utilizzati per configurare i modelli di licenza.

Per un esempio-to-end che utilizza le classi .NET per configurare il modello di licenza PlayReady, vedere [uso PlayReady dinamico crittografia e il servizio di recapito licenza](https://msdn.microsoft.com/library/azure/dn783467.aspx).

##<a id="classes"></a>Classi .NET servizi multimediali che vengono utilizzate per configurare i modelli di licenza

Di seguito sono che le classi .NET principale vengono utilizzate per configurare i modelli di licenza PlayReady di servizi di supporto. Queste classi mappare a definiti nello [schema XML del modello di licenza PlayReady](media-services-playready-license-template-overview.md#schema).

La classe [MediaServicesLicenseTemplateSerializer](https://msdn.microsoft.com/library/azure/microsoft.windowsazure.mediaservices.client.contentkeyauthorization.mediaserviceslicensetemplateserializer.aspx) viene utilizzata per serializzare e deserializzare da e verso il modello di licenza di servizi multimediali XML.

###<a name="playreadylicenseresponsetemplate"></a>PlayReadyLicenseResponseTemplate

[PlayReadyLicenseResponseTemplate](https://msdn.microsoft.com/library/azure/microsoft.windowsazure.mediaservices.client.contentkeyauthorization.playreadylicenseresponsetemplate.aspx) - questa classe rappresenta il modello per la risposta inviata all'utente finale. La presentazione contiene un campo in una stringa di dati personalizzati tra il server licenze e l'applicazione (può essere utile per la logica app personalizzato), nonché un elenco di uno o più modelli di licenza.

Questa è la classe "livello superiore" nella gerarchia del modello. Indicare che il modello di risposta include un elenco dei modelli di licenza e i modelli di licenza includono (direttamente o indirettamente) tutte classi che costituiscono i dati del modello da serializzare.


###<a name="playreadylicensetemplate"></a>PlayReadyLicenseTemplate

[PlayReadyLicenseTemplate](https://msdn.microsoft.com/library/azure/microsoft.windowsazure.mediaservices.client.contentkeyauthorization.playreadylicensetemplate.aspx) - la classe rappresenta un modello di licenza per la creazione di licenze PlayReady da restituire agli utenti finali. Contiene i dati nella chiave del contenuto alla licenza e diritti o limitazioni da applicare dal runtime PlayReady DRM quando si usa la chiave del contenuto.


###<a id="PlayReadyPlayRight"></a>PlayReadyPlayRight

[PlayReadyPlayRight](https://msdn.microsoft.com/library/azure/microsoft.windowsazure.mediaservices.client.contentkeyauthorization.playreadyplayright.aspx) - questa classe rappresenta PlayRight di una licenza PlayReady. Consente all'utente la possibilità di riproduzione del contenuto con le limitazioni zero o più configurato nella licenza e su PlayRight (per i criteri specifici di riproduzione). La maggior parte dei criteri nel PlayRight deve fare con restrizioni di output che consentono di controllare i tipi di output che è possibile riprodurre il contenuto tramite e le eventuali limitazioni che devono essere inseriti in posizione quando si utilizza un output specificato. Ad esempio, se è abilitata la DigitalVideoOnlyContentRestriction, quindi runtime DRM consentirà solo il video deve essere presentata su uscite digitali (analogici uscite video non possono essere passare il contenuto).

>[AZURE.IMPORTANT]I tipi di restrizioni possono essere molto potenti ma possono influire sulle esperienza utente. Se le protezioni output sono configurate restrittive, il contenuto potrebbe essere inutilizzabile in alcuni client. Per ulteriori informazioni, vedere il documento [PlayReady conformità regole](https://www.microsoft.com/playready/licensing/compliance/) .

Per un esempio di protezione quali livelli supporta Silverlight, vedere: [Silverlight supporta per protezioni output](http://go.microsoft.com/fwlink/?LinkId=617318).
