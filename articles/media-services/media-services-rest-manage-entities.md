#<a name="managing-media-services-entities-with-rest-api"></a>Gestione dei supporti servizi entità con API REST

> [AZURE.SELECTOR]
- [RESTO](media-services-rest-manage-entities.md)
- [.NET](media-services-dotnet-manage-entities.md)

Servizi multimediali di Microsoft Azure è un servizio basato su resto basato su OData v3. Per questi motivi, è possibile aggiungere, eseguire una query, aggiornare ed entità nello stesso modo è eliminare come in qualsiasi altro servizio di OData. Eccezioni verranno evidenziate quando applicabile. Per ulteriori informazioni sull'OData, vedere [la documentazione Open Data Protocol](http://www.odata.org/documentation/).

- Aggiunta di entità 
- Query su entità 
- Enumerazione di raccolte di grandi dimensioni di entità
- Aggiornamento di entità 
- Eliminazione di entità 

>[AZURE.NOTE] Quando si utilizza l'API REST di servizi multimediali, le considerazioni seguenti:
>
>Quando si accede a entità in servizi di supporto, è necessario impostare i campi specifici dell'intestazione e i valori nelle convocazioni HTTP. Per ulteriori informazioni, vedere [il programma di installazione per lo sviluppo di API REST servizi multimediali](media-services-rest-how-to-use.md).

>Dopo aver completato la connessione a https://media.windows.net, si riceverà un reindirizzamento 301 specificando un altro URI di servizi di supporto. È necessario effettuare chiamate successive al nuovo URI come descritto nella [connessione ai servizi multimediali utilizzando l'API REST](media-services-rest-connect-programmatically.md). 


##<a name="adding-entities"></a>Aggiunta di entità

Ogni entità in servizi multimediali di viene aggiunto a un set di entità, ad esempio risorse, tramite una richiesta HTTP POST.

Nell'esempio seguente viene illustrato come creare un AccessPolicy.

    POST https://media.windows.net/API/AccessPolicies HTTP/1.1
    Content-Type: application/json;odata=verbose
    Accept: application/json;odata=verbose
    DataServiceVersion: 3.0
    MaxDataServiceVersion: 3.0
    x-ms-version: 2.11
    Authorization: Bearer http%3a%2f%2fschemas.xmlsoap.org%2fws%2f2005%2f05%2fidentity%2fclaims%2fnameidentifier=youraccountname&urn%3aSubscriptionId=2f84471d-b1ae-4e75-aa09-010f0fc0cf5b&http%3a%2f%2fschemas.microsoft.com%2faccesscontrolservice%2f2010%2f07%2fclaims%2fidentityprovider=https%3a%2f%2fwamsprodglobal001acs.accesscontrol.windows.net%2f&Audience=urn%3aWindowsAzureMediaServices&ExpiresOn=1337067658&Issuer=https%3a%2f%2fwamsprodglobal001acs.accesscontrol.windows.net%2f&HMACSHA256=dithjGvlXR9HlyAf5DE99N5OCYkPAxsHIcsTSjm9%2fVE%3d
    Host: media.windows.net
    Content-Length: 74
    Expect: 100-continue
    
    {"Name": "DownloadPolicy", "DurationInMinutes" : "300", "Permissions" : 1}

##<a name="enumerating-through-large-collections-of-entities"></a>Enumerazione di raccolte di grandi dimensioni di entità

Quando si eseguono query entità, non esiste un limite di 1000 entità restituite contemporaneamente perché pubblico resto v2 limita i risultati della query ai risultati di 1000. Consente di enumerare vasta gamma di entità **ignorare** e **margine superiore** . 

Nell'esempio seguente viene illustrato come utilizzare **ignorare** e **margine superiore** per ignorare i processi prima di tutto 2000 e svolgere il lavoro successivo 1000.  

    GET https://media.windows.net/api/Jobs()?$skip=2000&$top=1000 HTTP/1.1
    Content-Type: application/json;odata=verbose
    Accept: application/json;odata=verbose
    DataServiceVersion: 3.0
    MaxDataServiceVersion: 3.0
    x-ms-version: 2.11
    Authorization: Bearer http%3a%2f%2fschemas.xmlsoap.org%2fws%2f2005%2f05%2fidentity%2fclaims%2fnameidentifier=youraccountname&urn%3aSubscriptionId=2f84471d-b1ae-4e75-aa09-010f0fc0cf5b&http%3a%2f%2fschemas.microsoft.com%2faccesscontrolservice%2f2010%2f07%2fclaims%2fidentityprovider=https%3a%2f%2fwamsprodglobal001acs.accesscontrol.windows.net%2f&Audience=urn%3aWindowsAzureMediaServices&ExpiresOn=1337078831&Issuer=https%3a%2f%2fwamsprodglobal001acs.accesscontrol.windows.net%2f&HMACSHA256=suFkxhvPWxQVMjOYelOJfYEWkyTWJCBc02pF0N7NghI%3d

##<a name="deleting-entities"></a>Eliminazione di entità

Entità possono essere eliminate in servizi multimediali mediante una richiesta HTTP eliminare. A seconda dell'entità, l'ordine in cui si elimina entità potrebbe essere importante. Ad esempio, è necessario che entità, ad esempio risorse di revocare (o eliminazione) Locator tutti che fanno riferimento a quel particolare attività prima di eliminare la risorsa.

Nell'esempio seguente viene illustrato come eliminare un indicatore di posizione utilizzato per caricare un file in archiviazione blob.

    DELETE https://media.windows.net/API/Locators('nb:lid:UUID:76dcc8e8-4230-463d-97b0-ce25c41b5c8d') HTTP/1.1
    Content-Type: application/json;odata=verbose
    Accept: application/json;odata=verbose

    MaxDataServiceVersion: 3.0
